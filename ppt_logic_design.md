- 项目名称：智能简历生成系统 / AI Resume Assistant（待定）
- 副标题：基于 RAG 与语义分析的个性化简历优化平台

### 第 2 页：研究背景 - 社会需求
**逻辑**：从宏观政策/社会趋势切入
- 就业市场竞争激烈（2024 届毕业生 1179 万，创历史新高）
- 简历是求职第一关，但大多数人不会写简历
- AI 辅助求职成为刚需（政策：数字经济、AI+ 行动）
- 数据支撑：XX% 的简历因格式/内容问题被 ATS 系统过滤

### 第 3 页：研究背景 - 真实痛点
**逻辑**：用具体案例凸显问题严重性（类似交通事故案例）
- 案例 1：应届生简历石沉大海（缺乏针对性）
- 案例 2：转行者简历被 ATS 过滤（关键词不匹配）
- 案例 3：经验丰富者简历冗长（信息过载）
- 配图：简历被拒的截图 / ATS 系统界面

### 第 4 页：研究对象
**逻辑**：明确要解决什么问题
- 研究对象：求职者的简历优化全流程
- 核心目标：从"通用模板"到"个性化定制"
- 技术路径：AI 理解用户背景 → 检索优质案例 → 生成针对性简历 → 语义评分优化

## ========== 第二部分：现状分析（别人怎么做，有什么问题）==========

### 第 5 页：现有方案的局限
**逻辑**：对比"传统方案"和"我们的方案"（类似事后检测 vs 事前预警）

| 维度 | 传统简历工具 | 我们的方案 |
|------|------------|----------|
| 生成方式 | 静态模板填空 | AI 动态生成 |
| 个性化 | 无针对性 | 基于职位语义匹配 |
| 质量保障 | 人工检查 | RAG 检索 + ATS 评分 |
| 交互方式 | 一次性生成 | 追问式对话 + 流式反馈 |

配图：左边是传统工具截图，右边是你的系统截图

### 第 6 页：技术瓶颈
**逻辑**：现有 AI 简历工具的三大痛点（类似监督学习的三大痛点）

1. **缺乏领域知识**：通用 LLM 不了解行业简历规范
2. **黑盒生成**：无法解释为什么这样写，用户不信任
3. **静态评估**：无法量化简历与职位的匹配度

配图：三个图标 + 简短说明

## ========== 第三部分：我们的方案（我们怎么做得更好）==========

### 第 7 页：核心创新点
**逻辑**：三大技术突破（类似无监督 + 多模态 + 轻量化）

1. **RAG 检索增强**：从优质简历库中检索相似案例，注入领域知识
2. **语义 ATS 评分**：基于 Embedding 的职位匹配度量化分析
3. **结构化输出**：Pydantic + Instructor 保证生成质量，可解释可编辑
4. **多模态输入**：支持上传 PDF/PPT 辅助材料，提取项目细节

配图：四个模块的图标 + 关键词

## ========== 第四部分：技术实现（怎么实现的）==========

### 第 8 页：整体架构图
**逻辑**：完整技术链路（类似四模块流程图）

```
用户输入 → 追问式对话 → RAG 检索 → 结构化生成 → 语义 ATS 评分 → 流式输出 → 用户修改
```

配图：流程图，每个模块标注技术栈

### 第 9 页：模块 1 - 追问式对话与记忆机制
**逻辑**：如何理解用户需求

- 技术：Profile Memory Service（4KB 压缩记忆）
- 功能：信息不足时主动追问，避免生成空洞内容
- 代码示例：`ProfileMemoryService` 的核心逻辑

配图：对话流程图 + 代码片段

### 第 10 页：模块 2 - RAG 检索增强生成
**逻辑**：如何注入领域知识

- 技术：Embedding Service + ChromaDB / Local Vector Store
- 数据：中英文参考简历库（`sample_reference_resumes.json`）
- 流程：用户输入 → Embedding → Top-K 检索 → 注入 Prompt

配图：RAG 流程图 + 检索结果示例

### 第 11 页：模块 3 - 结构化输出与契约验证
**逻辑**：如何保证生成质量

- 技术：Instructor + Pydantic Schema
- 优势：强制 LLM 输出符合 `StructuredResume` 契约
- 对比：传统 JSON 解析 vs 结构化输出

配图：代码对比 + Schema 示例

### 第 12 页：模块 4 - 语义 ATS 评分
**逻辑**：如何量化简历质量

- 技术：Semantic ATS Service（Embedding 相似度计算）
- 功能：简历与职位描述的语义匹配度
- 可视化：雷达图展示各维度得分

配图：ATS Dashboard 截图 + 雷达图

### 第 13 页：模块 5 - 流式生成与实时反馈
**逻辑**：如何提升用户体验

- 技术：SSE (Server-Sent Events) 流式传输
- 优势：实时看到生成过程，降低等待焦虑
- 对比：传统等待 vs 流式生成

配图：流式生成的动图 / 截图对比
